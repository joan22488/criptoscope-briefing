01
Visión general
Un sistema que monitoriza el mercado cripto 24/7, genera análisis con IA y publica automáticamente en Telegram y X.

CriptoScope es una plataforma completa que combina datos de mercado en tiempo real, análisis técnico multi-timeframe y generación de contenido con inteligencia artificial. El sistema opera de forma autónoma en la nube y puede controlarse íntegramente desde Telegram.

Qué hace el sistema

07:00 diario
Briefing matinal completo — análisis de mercado, noticias destacadas, Fear&Greed, señales técnicas y pregunta para la comunidad. Se publica en Telegram y X como thread.
07 · 11 · 15 · 19h
Señales técnicas BTC/ETH/SOL — análisis top-down 1D→4H→1H→15m con RSI, MACD, EMA, pivots y funding rate. LONG / SHORT / ESPERAR con entrada, TP y SL.
Cada 5 min
Monitor de alertas de precio — verifica los niveles guardados con /alerta y notifica en privado cuando se alcanzan.
Cada 15 min
Monitor de noticias RSS — revisa CoinDesk. Si detecta una noticia con tus keywords la envía en privado con botones para publicar al instante.
Cada 30 min
Alertas de eventos macro y verificación de señales pendientes — backtesting automático contra precio real.
Domingos 09:00
Resumen semanal — estadísticas de señales, aciertos/fallos, contexto de la semana y outlook para los próximos días.
Bajo demanda
Bot de Telegram — 16 comandos para publicar contenido, consultas privadas, alertas de precio, encuestas y publicaciones programadas.
02
Arquitectura
Módulos independientes en Node.js ES Modules. Cada fichero tiene una responsabilidad única.
index.js
Punto de entrada. Arranca todos los crons y el bot en paralelo.
bot.js
Bot de Telegram. Long-polling, 16 comandos, callbacks de botones.
pipeline.js
Orquesta el briefing matinal: reúne datos, llama a Claude, publica.
signals.js
Análisis técnico top-down. Descarga velas de OKX, calcula indicadores, genera señal con Claude.
coindesk.js
Precios, Fear&Greed, dominancia BTC, gainers/losers vía CoinGecko. Noticias vía RSS.
weekly.js
Resumen semanal con estadísticas de señales del tracker.
alerts.js
Monitor de eventos macro de alto impacto (ForexFactory).
tracker.js
Registro y backtesting de señales. Guarda en Notion o en signals.json.
calendar.js
Calendario económico ForexFactory — CPI, NFP, FOMC, Fed.
notion.js
Integración Notion v5 — briefings y señales en bases de datos.
telegram.js
Envío chunked a Telegram. Split en párrafos → líneas → caracteres.
twitter-post.js
Threads en X vía OAuth 1.0a. CTA de Telegram al final de cada hilo.
twitter.js
Lectura de tweets de cuentas clave vía Nitter RSS (contexto para Claude).
reddit.js
Señales de comunidad vía Hacker News API.
prompts.js
Voz editorial CriptoScope. Plantillas JSON para todos los módulos.
output.js
Guardado local de briefings en archivos.

Despliegue: Railway Cloud (Europe West). Sin puertos expuestos. Restart automático en fallo. El bot corre en long-polling concurrente con los crons mediante iniciarBot() no bloqueante.

03
Publicación automática
Todo lo que el sistema publica sin intervención manual.

Briefing matinal (07:00)

El pipeline recoge en paralelo: precios BTC/ETH/SOL, Fear&Greed Index, dominancia BTC, top gainers/losers, liquidaciones 24h, funding rate, noticias de CoinDesk, tweets de cuentas clave, señales de Hacker News y calendario macro. Claude genera un paquete editorial completo con voz de trader: briefing, thread de 5 tweets para X, guion de vídeo y pregunta para la comunidad. Se publica en Telegram con HTML enriquecido y en X como thread real encadenado.

Señales técnicas (07 · 11 · 15 · 19h)

Análisis top-down de BTC, ETH y SOL. Se descargan 120 velas de OKX en cada timeframe (1D, 4H, 1H, 15m), se calculan todos los indicadores y Claude genera una señal con entrada, TP1, TP2, SL y ratio R:R. Las señales con entrada definida se registran automáticamente para backtesting.

Backtesting automático (cada 30 min)

Cada señal registrada se verifica contra el precio real. Al tocar TP1, TP2 o SL el sistema lo registra en Notion y envía el resultado al canal. Las señales expiran a las 48 horas.

Resumen semanal (domingos 09:00)

Estadísticas de la semana: señales emitidas, tasa de acierto, mejor y peor trade, análisis del mercado de la semana y outlook para los próximos días.

Control de pausa: todos los crons respetan isPausado(). Con /pausa se detienen todas las publicaciones automáticas sin apagar el servidor. El bot sigue respondiendo comandos privados.

Análisis de fotos (sin comando)

Manda una captura de pantalla de cualquier noticia al bot. Claude verifica la credibilidad (VERIFICADA / PROBABLE / DUDOSA / FALSA), genera un análisis al estilo CriptoScope y te ofrece publicarlo en canal + X o guardarlo solo para ti. Si mandas una foto de un comentario con el pie "responde", Claude te redacta una respuesta directa sin publicarla.

Monitor automático de noticias

Cada 15 minutos el sistema revisa CoinDesk RSS. Cuando detecta una noticia con tus keywords configuradas, te la envía en privado con tres botones: ⚡ Publicar flash, 📝 Hacer hilo o 🙈 Ignorar. Un clic y el contenido se genera y publica sin escribir nada más.

Análisis técnico
Metodología top-down multi-timeframe implementada en signals.js.

Metodología top-down

TimeframeRolIndicadores
1DFiltra la tendencia macro. Si el Daily está bajista, no se buscan LONGs.RSI 14, EMA 20/50, MACD, divergencias
4HConfirma estructura. Pivots calculados aquí (R1, R2, S1, S2).RSI 14, EMA 20/50, MACD, niveles pivot
1HValida RSI/MACD. Las divergencias en 1H/4H reducen el tamaño de posición.RSI 14, MACD 12/26/9, divergencias RSI y MACD
15mGatillo de entrada. Precio de referencia para la señal.RSI 14, EMA 20/50, soporte/resistencia local

Las velas provienen de OKX API pública (/api/v5/market/candles). Binance devuelve HTTP 451 desde Railway (restricción geográfica). OKX no tiene restricciones y su API es gratuita sin necesidad de cuenta.

CoinGecko Demo API: 10.000 créditos/mes. El sistema usa aproximadamente 64 créditos/mes con 3 llamadas paralelas en el briefing matinal. Límite de velocidad: 100 llamadas/minuto. Requiere header x-cg-demo-api-key.

Notas técnicas

OKX devuelve velas newest-first. Se aplica .reverse() para obtener orden cronológico antes de calcular indicadores.

Parser JSON de dos capas. Claude a veces devuelve JSON con formato incorrecto. El sistema extrae primero el bloque {...} y si falla aplica regex campo a campo.

Chunking de Telegram. Los mensajes se dividen en trozos de máximo 4000 caracteres respetando primero párrafos (\n\n), luego líneas (\n) y por caracteres como último recurso.

Notion SDK v5. La versión 5 eliminó databases.query(). Se usa notion.request() directamente contra la REST API.

Instalación y despliegue
Pasos para montar el sistema desde cero o republicarlo en un nuevo entorno.

1. Preparar el entorno local

PasoComando
Clonar repogit clone <url> && cd criptoscope-briefing
Instalar dependenciasnpm install
Crear .envcp .env.example .env y rellenar las variables
Probar briefingnpm run once
Probar señalesnpm run signals
Modo producciónnpm start (crons + bot)

2. Crear el bot de Telegram

Busca @BotFather en Telegram → /newbot → elige nombre y username → copia el token en TELEGRAM_BOT_TOKEN. Crea el canal, añade el bot como administrador con permisos de publicar. Para obtener el ID del canal: reenvía un mensaje del canal a @userinfobot. Para tu ID personal: escríbele directamente a @userinfobot.

3. Despliegue en Railway

PasoComando
Instalar CLInpm install -g @railway/cli
Loginrailway login
Vincular proyectorailway link → seleccionar proyecto
Subir variablesrailway variables set KEY=value
Desplegarrailway up --service <nombre-servicio>

Configuración recomendada Railway: Region Europe West · Restart policy: On failure · Sin puertos expuestos. Para despliegues futuros basta con ejecutar railway up --service adaptable-happiness desde la carpeta del proyecto.
